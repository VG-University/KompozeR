\section{Conclusioni}

KompozeR e' stato concepito come un'estensione evoluta dell'e-commerce Kompo, con l'obiettivo di rendere piu' semplice e guidata la configurazione e l'acquisto di scaffalature personalizzabili. 
Il progetto affronta il problema sia dal punto di vista dell'esperienza utente, introducendo un configuratore visuale, un chatbot di supporto e notifiche contestuali, sia dal punto di vista dell'organizzazione software, tramite una struttura modulare basata su servizi distinti.

\subsection{Risultati raggiunti}

I principali risultati ottenuti sono:

\begin{itemize}
	\item definizione dei requisiti e del dominio applicativo;
	\item progettazione di un'architettura web coerente con il problema affrontato;
	\item individuazione di servizi separati per autenticazione, catalogo, configurazione, carrello, notifiche, chatbot e reporting;
	\item implementazione di un API Gateway con autenticazione JWT centralizzata e propagazione identita' verso i servizi;
	\item realizzazione del flusso CAD completo con finalizzazione e sincronizzazione automatica del carrello;
	\item implementazione di notifiche cross-service e reportistica trend ordini in area amministrativa;
	\item definizione di contratti applicativi chiari per API REST, messaggi realtime e DTO di dominio.
\end{itemize}

\subsection{Limiti attuali}

I limiti principali della versione attuale sono:

\begin{itemize}
	\item assenza di una campagna formale di user testing su campione esteso;
	\item metriche prestazionali e di observability non ancora consolidate in modo sistematico nel report;
	\item alcuni aspetti di hardening operativo (pipeline CI/CD complete, policy di deploy multi-ambiente).
\end{itemize}

\subsection{Sviluppi futuri}

Le evoluzioni naturali del progetto includono:

\begin{itemize}
	\item estensione della copertura test automatizzata e introduzione di metriche di quality gate piu' stringenti;
	\item miglioramento UX del configuratore e dei meccanismi di feedback in tempo reale (con un'interfaccia più competitiva anche esteticamente);
	\item consolidamento del deployment verso ambienti diversi da quello locale;
	\item introduzione di un LLM per migliorare l'interazione con il chatbot e fornire suggerimenti più intelligenti agli utenti.
\end{itemize}
