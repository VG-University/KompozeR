\section{Tecnologie}

Per il progetto e' stato adottato lo stack \textbf{MEVN} esteso a microservizi (MongoDB, Express, Vue, Node.js), con componenti aggiuntivi per realtime e integrazione eventi.

Le tecnologie utilizzate per sviluppare il sistema sono:

\begin{itemize}
	\item \textbf{Node.js} ed \textbf{Express}: sviluppo dei servizi backend e dell'API Gateway;
	\item \textbf{Vue 3}: sviluppo componente client (SPA);
	\item \textbf{MongoDB} + \textbf{Mongoose}: persistenza dati per i diversi contesti applicativi;
	\item \textbf{Socket.IO}: comunicazione realtime per notifiche e chatbot;
	\item \textbf{Redis}: broker in-memory per eventi publish/subscribe tra servizi;
	\item \textbf{Docker} + \textbf{Docker Compose}: esecuzione e deploy locale dell'intero sistema.
\end{itemize}

I linguaggi e le tecnologie di sviluppo usate nel progetto sono:

\begin{itemize}
	\item \textbf{TypeScript}: sia lato client sia lato server;
	\item \textbf{HTML}: struttura delle pagine tramite template Vue;
	\item \textbf{CSS}: definizione stile e layout delle interfacce.
\end{itemize}

\subsection{Docker}

Docker rappresenta lo standard de-facto per la distribuzione di sistemi multi-componente. 
Nel progetto KompozeR l'applicazione viene avviata tramite \texttt{docker-compose}, che orchestra frontend, API Gateway, microservizi backend, MongoDB e Redis.

Nel file di compose sono presenti tre gruppi principali di servizi:

\begin{itemize}
	\item \textbf{infrastruttura}: \texttt{redis} e istanze Mongo dedicate (auth, catalog, cad, cart, order, notification, chat);
	\item \textbf{backend}: \texttt{auth-service}, \texttt{catalog-service}, \texttt{cad-service}, \texttt{cart-service}, \texttt{order-service}, \texttt{notification-service}, \texttt{chatbot-service}, \texttt{reporting-service}, \texttt{api-gateway};
	\item \textbf{frontend}: servizio \texttt{frontend} (Vite).
\end{itemize}

Nel caso di KompozeR il ruolo di "entry point" e' svolto da \texttt{api-gateway}, che centralizza autenticazione JWT, routing delle chiamate HTTP e gestione degli handshake realtime verso i servizi applicativi.

\subsection{WebSocket}

WebSocket e' la tecnologia che consente comunicazione bidirezionale e persistente tra client e server. 
Nel progetto e' implementata tramite Socket.IO per due funzionalita' centrali:

\begin{itemize}
	\item \textbf{notifiche realtime}: push immediato di eventi utente (es. variazioni di disponibilita'/prezzo rilevanti);
	\item \textbf{chatbot}: scambio messaggi utente-bot e indicatore di typing in tempo quasi reale.
\end{itemize}

Rispetto al polling periodico, questo approccio riduce il numero di richieste inutili e migliora la latenza percepita dall'utente, perche' l'aggiornamento avviene appena l'evento e' disponibile.

\subsection{Redis}

Redis e' usato come componente in-memory per la comunicazione asincrona tra servizi (pub/sub), in particolare per propagare eventi di catalogo verso i moduli che devono reagire in modo disaccoppiato (ad esempio notifiche e riallineamento carrello).
